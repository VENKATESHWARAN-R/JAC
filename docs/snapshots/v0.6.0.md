## What changed in v0.6.0

### 1. Workspace loose-mode

**Problem:** running `jac` outside a git repo created `.agents/` next to whatever files
happened to be there — surprising and messy.

**Fix:** JAC now distinguishes a *project* from a *loose* run.

- A **project** is any directory with `.git` (file or dir, git worktrees work)
  or `.agents/` — the new explicit opt-in.
- A **loose** run has neither. Session state (sessions, `usage.jsonl`, A2A)
  anchors to `~/.jac` rather than the CWD.

```
# you are in ~/tmp — no .git, no .agents/
jac                   # sessions go to ~/.jac/sessions/; project memory refused
jac init              # offers to create .agents/ here; accept to make it a project
```

Key path functions:

| Function | Returns | Notes |
|---|---|---|
| `project_root()` | `Path \| None` | `None` when loose |
| `find_project_root()` | `Path` | CWD fallback — for tool execution |
| `project_state_root()` | `Path` | Where JAC writes its state; `~/.jac` when loose |
| `in_project()` | `bool` | Replaces `is_in_project_repo()` |

### 2. Session management

**List:**
```bash
jac sessions          # oldest → newest; shows message count + creation time
jac sessions list     # explicit subcommand form
```

**Delete:**
```bash
jac sessions delete 2026-05-29T10-00-00          # prompts for confirmation
jac sessions delete 2026-05-29T10-00-00 --yes    # skip prompt

# from the REPL (refuses to delete the active session):
/sessions delete 2026-05-29T10-00-00
```

**Prune by age:**
```bash
jac sessions prune --older-than 30d              # dry-run: shows what would be deleted
jac sessions prune --older-than 30d --yes        # actually deletes

# from the REPL:
/sessions prune 30d          # dry-run preview
/sessions prune 30d yes      # confirm inline (no blocking prompt in REPL)
```

Supported duration units: `w` (weeks), `d` (days), `h` (hours). Examples: `2w`, `30d`, `12h`.

Deletion removes the session directory (messages, plan, compacted slices) but
leaves `usage.jsonl` intact — those tokens were spent and still count toward
project totals.

**Richer listing:** sessions now show message count and human-readable creation
time, not just bare IDs. An unreadable `messages.json` keeps the session listed
with an `unreadable` count.

**Atomic saves:** `Session.save()` writes via a sibling tempfile + rename. A
kill mid-write can no longer produce a truncated `messages.json`.

### 3. Memory slash commands

Three new slash commands let you inspect and edit memory directly, without
waiting for Gru to decide:

**`/memory [user|project]`** — read-only view of stored entries:
```
/memory            # both scopes
/memory user       # ~/.jac/memory.md only
/memory project    # <repo>/.agents/memory.md only
```
Output strips audit comments so the prose can be copied back into a `forget`
call. Reports "not created yet" or "empty" cleanly.

**`/remember <scope> <category> <text>`** — write a memory entry directly:
```
/remember project convention uses uv, not pip
/remember user preference prefers terse output
```
Valid scopes: `user`, `project`. Valid categories: `convention`, `fact`,
`preference`, `gotcha`, `decision`. Same audited single-bullet format as the
agent-initiated `remember` tool; de-duplicates before writing.

**`/forget <scope> <text>`** — remove an entry:
```
/forget project uses uv, not pip
```
Exact-normalised match (same rules as the agent tool). Refuses if the text is
ambiguous or not found.

These bypass the HITL approval panel — the typed command is the approval,
because the human is already in the loop.

### 4. Minion renderer polish

**Color grammar — now enforced.** The rule is: yellow = Gru, blue = minion.
Previously `SubAgentQuestion` (a message *from* the minion) used a yellow
border, and approval panels from a minion used a yellow border with blue text
inside. Now:

| Event | Border before | Border now |
|---|---|---|
| `SubAgentSpawned` | blue | blue ✓ |
| `SubAgentQuestion` (from minion) | yellow | **blue** |
| `SubAgentAnswer` (from Gru) | cyan | **yellow** |
| Approval — minion | yellow | **blue** |
| Approval — Gru | yellow | yellow ✓ |

**Label consistency.** Panel titles unified: "sub-agent" replaced with "minion"
everywhere (`▶ minion`, `⏸ minion → Gru`, `↩ Gru → minion`, `✓ minion`).
The IDs already said `minion-N`; labels now match.

**Status bar redesign:**

- Dark background (`noreverse bg:ansiblack`) — no more white-on-dark-terminal clash.
- Key labels (`profile:`, `tier:`, `ctx:`, …) now `ansiyellow` — warm against
  the dark canvas, on-theme with Gru's colour.
- Values explicit `ansiwhite`; secondary info (model short-name, session ID) `ansibrightblack`.
- Segments joined with ` · ` in `ansibrightblack` instead of raw spaces.
- `spawns:N` renamed to `minions:N`.

**Thinking-label pool** expanded from 4 to 20+ minion-adjacent phrases
(`bello…`, `bee do bee do…`, `gelato…`, `banana…`, `poulet tikka masala…`, …).

## Upgrade notes

### `is_in_project_repo()` → `in_project()`

If you call `jac.workspace.paths.is_in_project_repo()` from custom code,
rename it to `in_project()`. The semantics are identical except it also
recognises `.agents/` as a project marker.

### `project_root()` return type

`project_root()` now returns `Path | None` (was always `Path`). If you relied
on the old "returns CWD as fallback" behaviour for *state paths*, switch to
`project_state_root()`. For *tool* paths (where files actually live), use
`find_project_root()` which retains the CWD fallback.

### Session listing

`Session.list_ids()` is unchanged. `Session.list_summaries()` is new and
returns `list[SessionSummary]` with `session_id`, `message_count`, and
`created`. The CLI listing now uses this for richer output.
